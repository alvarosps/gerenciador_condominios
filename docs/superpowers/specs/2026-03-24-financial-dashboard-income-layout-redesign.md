# Financial Dashboard — Income Section Layout Redesign

## Context

The financial dashboard (`/financial`) shows a summary of monthly income, expenses, and balance. The current layout has 4 cards in the top row (Entradas, Kitnets Não Alugados, Despesas, Saldo) and a 5-column grid "Resumo de Entradas" section below with "Outras Entradas" mixed in.

This redesign reorganizes the income section for better clarity, visual hierarchy, and responsiveness.

## Design Decisions

### 1. "Total Entradas" includes ALL income

The "Total Entradas no Mês" card shows the total revenue generated by the condominium — including owner apartments (Tiago, Alvaro) + condominium apartments + extra incomes. This is the full picture of what the business generates.

**However**, the "Saldo" continues to use only condominium income + extra incomes minus expenses, because Tiago/Alvaro rents go directly to the owners and never pass through the condominium's cash.

### 2. Three-row layout

**Row 1 — 3 summary cards (equal width):**
- **Total Entradas no Mês — {month}**: All income (kitnets + extra incomes). Subtexts list the breakdown (X kitnets: R$ Y, extra income items).
- **Despesas — {month}**: Monthly expenses + overdue total. Same as current.
- **Saldo — {month}**: Balance = condominium income + extra incomes − expenses − overdue. Same calculation as current.

**Row 2 — Resumo de Entradas Condomínio (full width):**
- **Header**: Card title with building icon
- **Total block** (centered): "Total Entradas Mensal" with value, kitnet count
- **Arrow**: "▼ distribui-se em"
- **3 distribution cards** (equal width, side by side):
  - **Condomínio**: blue left border, percentage badge (top-right), value, kitnet count, "→ vai pro caixa"
  - **{Owner 1}**: amber left border, percentage badge, value, apartment list, "→ repasse direto"
  - **{Owner 2}**: purple left border, percentage badge, value, apartment list, "→ repasse direto"
  - Note: owner cards are dynamic — rendered from `income_summary.owner_incomes` array
- **Potential bar** (below, separated by dashed border):
  - Left: "{N} kitnets não alugados" in amber
  - Right: "+ R$ {value}" in amber
  - Stacked bar: solid gradient (rented portion) + dashed pattern (vacant portion)
  - Labels: "Alugados: R$ X" left, "Potencial total: R$ Y" right

**Row 3 — Outras Entradas (full width):**
- **Header**: Card title with plus-circle icon
- **Total value** in green
- **Items in horizontal row**: Each extra income as a sub-card with left green border
  - Shows description, person name (if any), value
  - Badge "mensal" for recurring items

### 3. Removed from top row

The "Kitnets Não Alugados" card is removed from the top row. This information now lives inside the "Resumo de Entradas" card as the potential bar — a more contextual placement that shows the relationship between rented income and vacant potential.

## Visual Design

### Colors
- **Condomínio**: `#3b82f6` (blue-500) — primary income, goes to cash
- **Owner cards**: Fixed palette by index: `['#f59e0b', '#a855f7', '#06b6d4', '#ec4899', '#14b8a6']` (amber, purple, cyan, pink, teal). Cycle if more than 5 owners.
- **Kitnets não alugados**: `#f59e0b` (amber-500) — warning/opportunity color
- **Total/Outras Entradas**: `#22c55e` (green-500) — positive income
- **Despesas**: `#f97316` (orange-500) — expense color
- **Saldo positive**: `#22c55e`, **negative**: `#ef4444` (red-500)

### Percentage badges
Each distribution card shows a percentage badge in the top-right corner:
- Background: color at 20% opacity (e.g., `bg-blue-500/20`)
- Text: full color
- Font: small, bold, rounded pill shape

### Potential bar
- Solid portion: gradient from blue to purple (representing all rented income)
- Vacant portion: diagonal stripe pattern with amber at low opacity, dashed border
- Height: ~14px with rounded corners

### Responsive behavior
- **Row 1**: 3 columns on lg, 1 column stacked on mobile
- **Row 2**: Full width always. Internal 3 distribution cards: side by side on md+, stacked on mobile
- **Row 3**: Full width always. Internal items: side by side on md+, stacked on mobile

## Data Requirements

No backend changes needed. The existing `dashboard_summary` API already returns all required data:

- `income_summary.total_monthly_income` — total from rented kitnets
- `income_summary.owner_incomes[]` — per-owner breakdown with person_name, total, apartments
- `income_summary.owner_total` — sum of owner incomes
- `income_summary.condominium_income` — income that goes to cash
- `income_summary.condominium_kitnet_count` — count of non-owner rented kitnets
- `income_summary.vacant_count`, `vacant_lost_rent` — vacant kitnet data
- `income_summary.extra_incomes[]`, `extra_income_total` — other income items
- `current_month_income`, `current_month_expenses`, `current_month_balance` — for top cards

### "Total Entradas" calculation (new)

Currently `current_month_income = condominium_income + extra_income_total`. The new "Total Entradas" card needs: `total_monthly_income + extra_income_total` (includes owner incomes).

**IMPORTANT**: Do NOT use `current_month_income` for the "Total Entradas" card — it excludes owner rents. Compute from `income_summary.total_monthly_income + income_summary.extra_income_total` directly.

This can be computed on the frontend from existing data — no API change required.
