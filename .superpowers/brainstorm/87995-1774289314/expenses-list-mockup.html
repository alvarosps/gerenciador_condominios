Estrutura de URL para pagina de detalhes

Como identificar cada despesa na URL?

A

Por slug

/financial/expenses/alvaro, /financial/expenses/electricity

URLs limpas e legíveis. Precisa de slug mapping no backend.

B

Por tipo + ID

/financial/expenses/person/3, /financial/expenses/utility/electricity

Explicito, facil de parsear. Mais verboso mas sem ambiguidade.

C

Por query params

/financial/expenses/details?type=person&id=3

Flexivel, nao precisa de rotas dinamicas. Menos clean na URL.
